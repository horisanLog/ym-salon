## 📋 プロジェクト概要

このプロジェクトは、Y'm 整体サロンの Web サイトの開発です。AI 姿勢診断 × 整体 × エステの総合サロンとしてのブランド確立を目指します。

### 技術スタック

- **フレームワーク**: Next.js 14 (Pages Router)
- **言語**: TypeScript
- **スタイリング**: Tailwind CSS v4
- **デプロイ**: Vercel
- **データ管理**: ローカル JSON ファイル（CMS 不使用）
- **状態管理**: React Hooks
- **フォーム**: React Hook Form + Zod
- **アイコン**: Lucide React
- **地図**: Google Maps API
- **予約システム**: LINE Bot 連携 + 電話予約 + フォーム予約

## 🎯 開発の基本方針

### 1. 設計ドキュメントの完全遵守

以下のドキュメントに記載された全ての仕様を厳密に守ること：

- **要件定義.md**: ビジネス要件、ターゲット、KPI、プレオープン戦略
- **情報設計\_サイトマップ.md**: サイト構造、ページ構成、導線設計
- **コンテンツ設計.md**: 各ページの詳細コンテンツ、文言、法規制対応
- **デザイン指針.md**: デザインシステム、カラー、タイポグラフィ、コンポーネント設計
- **データ設計.md**: データモデル、TypeScript 型定義、JSON スキーマ
- **ルーティング設計.md**: URL 構造、ページ骨子、コンポーネント構成

### 2. 品質基準

#### パフォーマンス

- **LCP（Largest Contentful Paint）**: 2.5 秒以内（特にモバイル重視）
- **FID（First Input Delay）**: 100ms 以内
- **CLS（Cumulative Layout Shift）**: 0.1 以下
- **モバイルページ速度スコア**: 85 以上（PageSpeed Insights）

#### アクセシビリティ

- **WCAG 2.1 AA 準拠**を目標
- キーボード操作対応必須
- スクリーンリーダー対応（aria-label、alt 属性の適切な設定）
- 色のコントラスト比 4.5:1 以上
- タッチ領域 44px 以上（モバイル重視）

#### SEO・ローカル SEO

- title/meta description 最適化（各ページ固有）
- 構造化データ対応（LocalBusiness、Service、Review、Article）
- OGP 設定（SNS シェア対応）
- セマンティック HTML（適切な見出し階層）
- 地域名（戸田市、蕨市、さいたま市南部）の自然な配置

#### レスポンシブデザイン

- **SP ファースト設計**: モバイルから設計し、デスクトップに拡張
- ブレークポイント: sm(640px), md(768px), lg(1024px), xl(1280px)
- 横スクロール禁止
- 画像は可変（w-full h-auto）
- 固定予約バー（SP 専用）

```typescript
// lib/data-loader.ts
import fs from "fs"
import path from "path"

export function loadData<T>(filename: string): T[] {
  const filePath = path.join(process.cwd(), "src/data", filename)
  const fileContents = fs.readFileSync(filePath, "utf8")
  return JSON.parse(fileContents)
}

// 使用例（getStaticPropsで使用）
export function getServices(): Service[] {
  return loadData<Service>("services.json")
}

export function getFeaturedReviews(limit: number = 6): Review[] {
  const reviews = loadData<Review>("reviews.json")
  return reviews.filter((r) => r.photoPermission).slice(0, limit)
}

export function getSalonInfo(): SalonInfo {
  const salonInfoArray = loadData<SalonInfo>("salon-info.json")
  return salonInfoArray[0] // 単一オブジェクトとして取得
}
```

## 📝 コーディング規約

### TypeScript

- 型定義は必須（`any`禁止）
- インターフェースは`interface`を使用
- オプショナルプロパティは`?`で明示
- 整体サロン特有の型（Service、Review、SalonInfo）を活用

### コンポーネント

- ページコンポーネントは Pages Router の規約に従う
- クライアント側の状態管理が必要な場合は React Hooks を使用
- 1 ファイル 1 コンポーネント
- ファイル名は PascalCase（例：`Hero.tsx`）
- 整体サロン特有のコンポーネント（ReservationCTA、GoogleMap）を活用

### CSS

- Tailwind CSS のユーティリティクラスを優先
- 整体サロン用カラーパレット（ベージュ・薄グリーン）を使用
- カスタム CSS は最小限に
- `@layer components`で共通スタイル定義

### 法規制対応

- 医療法・薬機法に配慮した表現を使用
- 「改善をサポート」「美しいラインを目指す」等の適切な表現
- 個人差がある旨の明記
